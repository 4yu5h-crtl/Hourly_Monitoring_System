## Directory Purposes

**server/src/controllers:**
- Purpose: Contains the core logic for the API. Each file handles requests for a specific resource (e.g., `entries`, `shifts`).
- Contains: Async functions processing `Request` and `Response` objects.
- Key files: `entriesController.ts`, `shiftsController.ts`.

**server/src/db:**
- Purpose: Manages the SQLite database connection and schema.
- Contains: Migration scripts, SQL schema definitions, and the connection utility.
- Key files: `connection.ts`, `schema.sql`, `migrate.ts`.

**src/components:**
- Purpose: Houses reusable React components used across multiple pages.
- Contains: Functional components (`.tsx` files).
- Key files: `HMSTable.tsx` (main data grid), `ShiftSelector.tsx`.

**src/lib:**
- Purpose: Provides utility functions and services for the frontend.
- Contains: API client, storage service, and shared logic.
- Key files: `api.ts` (network calls), `storage.ts` (service layer and caching).

**src/pages:**
- Purpose: Represents the main views of the application, typically mapped to routes.
- Contains: Large page-level components.
- Key files: `DataEntry.tsx`, `History.tsx`.

## Key File Locations

**Entry Points:**
- `src/main.tsx`: Frontend initialization.
- `server/src/server.ts`: Backend initialization.

**Configuration:**
- `server/src/db/schema.sql`: Database table structure.
- `tailwind.config.ts`: Tailwind CSS configuration.
- `vite.config.ts`: Vite build and development configuration.

**Core Logic:**
- `src/lib/storage.ts`: Orchestrates frontend data and state.
- `server/src/controllers/entriesController.ts`: Processes hourly entry updates.

**Testing:**
- `src/test/`: Contains frontend tests (e.g., `example.test.ts`, `setup.ts`).

## Naming Conventions

**Files:**
- React components/pages: PascalCase (e.g., `DataEntry.tsx`, `HMSTable.tsx`).
- Utilities/APIs: camelCase (e.g., `storage.ts`, `api.ts`).
- Server routes/controllers: camelCase (e.g., `entries.ts`, `entriesController.ts`).

**Directories:**
- Plural nouns, lowercase (e.g., `components`, `routes`, `pages`).

## Where to Add New Code

**New Feature:**
- Frontend: Create a new page in `src/pages/` or a component in `src/components/`.
- Backend: Add a new route in `server/src/routes/` and a corresponding controller in `server/src/controllers/`.

**New Component/Module:**
- Shared components: `src/components/`.
- UI primitives: `src/components/ui/` (follow Shadcn UI pattern).

**Utilities:**
- Shared frontend helpers: `src/lib/utils.ts`.
- Backend helpers: `server/src/utils/`.

## Special Directories

**.planning:**
- Purpose: Contains system analysis, planning, and documentation generated by AI tools.
- Generated: No (manually or AI-generated).
- Committed: Yes.

**server/dist:**
- Purpose: Compiled backend code (not shown in provided tree but typically exists).
- Generated: Yes.
- Committed: No.
